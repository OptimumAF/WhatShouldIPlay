# Feedback Loop

This repository uses a lightweight, repeatable feedback loop to prioritize roadmap work.

## Channels

- GitHub Discussions: open-ended product discussion and vote threads.
- `Roadmap Feedback` issue template: structured requests tied to milestones and impact.
- `Feature Request` issue template: concrete implementation proposals.

## Cadence

1. Collect feedback continuously via Discussions and issue templates.
2. Review new feedback weekly during triage.
3. Publish priority updates in `docs/roadmap.md` when milestone scope changes.

## Prioritization Signals

- Number of users affected and severity of pain.
- Delivery risk and dependency complexity.
- Alignment with active milestone goals.
- Security, reliability, or trust impact.

## Maintainer Checklist

1. Tag incoming feedback with `roadmap` and area labels.
2. Link feedback items to a roadmap milestone issue.
3. Close or defer with clear rationale and next review point.
