# Governance

## Project Model

WhatShouldIPlay is currently maintainer-led.

- The maintainer team reviews and merges pull requests.
- Architectural or breaking changes should be proposed via issue before implementation.

## Decision Making

- Default path: lightweight consensus in pull request discussion.
- If consensus is unclear, maintainers make the final decision based on:
  - user impact
  - operational risk
  - maintainability
  - project direction

## Maintainers

- [@OptimumAF](https://github.com/OptimumAF)

## Becoming a Maintainer

Regular contributors may be invited based on:

- sustained high-quality contributions
- constructive review participation
- reliability in following repository standards
